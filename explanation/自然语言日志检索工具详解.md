# 自然语言日志检索工具详解（query_log）

## 概述

`query_log` 是一个基于 MCP 的自然语言日志检索工具，让运维/业务人员用日常语言查询腾讯云 CLS 日志，无需学习 CQL/Lucene 查询语法。

该工具封装了 CLS MCP 的 `TextToSearchLogQuery` → `SearchLog` 两步调用链路，用户只需描述想查什么，工具自动完成查询语句生成和日志检索。

## 文件位置

- **工具实现**: [app/tools/query_log_tool.py](../app/tools/query_log_tool.py)
- **MCP 客户端**: [app/agent/mcp_client.py](../app/agent/mcp_client.py)

## 核心功能

### 1. 自然语言 → CQL 自动转换

调用 CLS MCP 的 `TextToSearchLogQuery` 工具，将自然语言问题自动转换为 CLS CQL 查询语句。

```
用户输入: "查询服务下线前5分钟的错误日志"
    ↓ TextToSearchLogQuery
CQL 输出: level:ERROR AND message:*下线* AND ...
```

### 2. 执行日志检索

将生成的 CQL 语句传给 `SearchLog` 工具执行，返回结构化日志结果。

### 3. 多格式兼容

自动兼容 MCP 工具返回的多种格式（字符串、`[{"text": "..."}]`、字典），统一提取文本内容。

## 调用参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `question` | str | 是 | 自然语言描述的查询需求，如"最近15分钟的错误日志" |
| `topic_id` | str | 否 | 日志主题ID，不传则使用 CLS MCP 默认配置 |
| `hours_ago` | int | 否 | 查询最近多少小时的日志，默认 1 小时 |

## 调用链路

```
query_log(question="最近15分钟的错误日志")
    │
    ├─ 1. 获取 MCP 客户端连接
    │
    ├─ 2. 调用 TextToSearchLogQuery(Text=question)
    │      → 返回 CQL: level:ERROR AND ...
    │
    ├─ 3. 调用 SearchLog(Query=cql, Limit=20)
    │      → 返回结构化日志列表
    │
    └─ 4. 格式化结果返回给 Agent
```

## 使用示例

### 在 RAG Agent 对话中

```
用户: "帮我查一下 data-sync-service 最近1小时的错误日志"
  → Agent 调用 query_log(
        question="data-sync-service最近1小时的错误日志",
        hours_ago=1
    )

用户: "服务下线前有哪些异常"
  → Agent 调用 query_log(
        question="服务下线前的异常日志"
    )

用户: "查询包含timeout的日志"
  → Agent 调用 query_log(
        question="包含timeout的日志",
        topic_id="topic-001"
    )
```

## 故障处理

| 异常类型 | 返回内容 |
|----------|----------|
| MCP 工具缺失 | `"CLS MCP 服务缺少必需工具: TextToSearchLogQuery/SearchLog"` |
| CQL 生成失败 | `"无法将问题转换为日志查询语句: '{question}'"` |
| 查询执行失败 | `"日志查询失败: {error}，请检查 CLS MCP 服务连接"` |

## 与直接使用 MCP 工具的区别

| 对比项 | query_log | 直接调用 TextToSearchLogQuery + SearchLog |
|--------|-----------|------------------------------------------|
| 调用次数 | 1 次 | 2 次（Agent 需两次往返） |
| 自然语言友好 | 是 | 需理解两个工具的职责 |
| 出错恢复 | 内置错误处理 | 依赖 Agent 自主判断 |
| 适合场景 | 常规日志查询 | 复杂多步日志分析 |

## 依赖

- 腾讯云 CLS MCP 服务（已配置在 `.env` 的 `MCP_CLS_URL`）
- MCP 服务端须提供 `TextToSearchLogQuery` 和 `SearchLog` 两个工具
